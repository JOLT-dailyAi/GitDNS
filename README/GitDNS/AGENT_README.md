# 📡 GitDNS — Navigation Index for AI Agents

This directory contains pre-generated navigation indexes for GitHub repositories.
Each subdirectory maps one repository and contains two files.

---

## Files in each repo folder

### `{owner}_{repo}.zone` — Your primary navigation file
A token-optimized DNS-style zone file. Use this first.

```
$ORIGIN https://raw.githubusercontent.com/{owner}/{repo}/{branch}/

.gitignore
README.md
src/index.js
src/components/Button.tsx
...
```

**How to use:**
1. Read the `$ORIGIN` line — this is the base URL for every file in the repo
2. When you need to inspect a file, combine: `$ORIGIN` + relative path = full raw URL
3. Use your fetch tool to retrieve only the files you actually need
4. Never fetch all files upfront — read the map, decide what you need, fetch only that

### `{owner}_{repo}.tree` — Human-readable structure (for context only)
A rich, formatted tree of the repository with file sizes, icons, tech stack, and stats.
Use this to orient yourself in an unfamiliar codebase — not as a fetch source.
Do not attempt to ingest this file fully. Use it to identify which directories
and files are relevant to your task, then resolve URLs via the `.zone` file.

---

## Navigation protocol

```
STEP 1 — Read the .zone file
          Establish $ORIGIN. Scan the flat file list.

STEP 2 — Identify relevant files
          Based on your task, determine which files you need to read.
          Use directory structure (slashes in paths) to infer hierarchy.

STEP 3 — Fetch on demand
          Construct URL: $ORIGIN + relative/path/to/file.ext
          Fetch only that file. Read it. Proceed.

STEP 4 — Repeat as needed
          Return to the .zone index whenever you need another file.
          Never re-read the entire zone file after step 1.
```

---

## What NOT to do

- Do not attempt to fetch every file in the zone index
- Do not treat the `.tree` file as a fetch source — it contains no URLs suitable for direct fetching
- Do not re-read the zone file repeatedly — load it once into context, navigate from memory
- Do not guess file contents — if you need a file, fetch it via its raw URL
